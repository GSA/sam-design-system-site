<h1>Layout Pattern</h1>
<p>The layout pattern is a set of components designed to enable rapid prototyping and development of new pages within the design system.</p>
<p>Like HTML elements, each component has a content model, or a set of components that can used within it, and contexts in which the component can be used, or a set of components where a component can be used as a child.</p>
<p>The layout pattern also includes a pattern for managing data and interactions. By conforming to the configuration of the patterns, new variations have been developed as quickly as in a day.</p>

<h3>SamPageNextComponent</h3>
<p>The <strong>sam-page-next</strong> component is the main wrapper component for the page. For its content model, it can take a <strong>sam-toolbar</strong>, <strong>sam-aside</strong>, and a <strong>sam-main</strong>.</p>
<p>When the component is instantiated, the sam-page-next component connects any interactions between the children in its content model. For example, the toggle button in the toolbar component triggers the toggle method on the aside component. This interaction is setup by the sam-page-next controller.</p>
<p><em>Content-model:</em></p>
<ul>
<li><strong>sam-toolbar</strong></li>
<li><strong>sam-aside</strong></li>
<li><strong>sam-main</strong></li>
</ul>
<p><em>Contexts in which this component can be used:</em></p>
<ul>
<li>Any Angular application</li>
</ul>
<h3>SamToolbarComponent</h3>
<p>The <strong>sam-toolbar</strong> component provides user interactions that apply to the entire page. Toggling the aside, saving the criteria and downloading the current results are examples of actions that are taken at the page level.</p>
<p><em>Content model:</em> none</p>
<p><em>Contexts in which this component can be used:</em></p>
<ul>
<li><strong>sam-page-next</strong></li>
</ul>
<h3>SamAsideComponent</h3>
<p>The <strong>sam-aside</strong> component provides a side navigation area for additional page controls or information. The side navigation can be opened or closed, and it collapses on smaller screens.</p>
<p><em>Content model:</em> TBD</p>
<p><em>Contexts in which this component can be used:</em></p>
<ul>
<li><strong>sam-page-next</strong></li>
</ul>
<h3>SamMainComponent</h3>
<p>The <strong>sam-main</strong> component is a container component for the content of the page.</p>
<p>It accepts a <strong>sam-page-title</strong>, <strong>sam-filter-drawer</strong> and <strong>sam-main-content</strong> component as its content model.</p>
<p><em>Content model:</em></p>
<ul>
<li><strong>sam-page-title</strong></li>
<li><strong>sam-filter-drawer</strong></li>
<li><strong>sam-main-content</strong></li>
</ul>
<p><em>Contexts in which this component can be used:</em></p>
<ul>
<li><strong>sam-page-next</strong></li>
</ul>
<h3>SamPageTitleComponent</h3>
<p>The <strong>sam-page-title</strong> component provides a style title for the page. It also allows for super text and additional information to be displayed above and below the title respectively.</p>
<p><em>Content model:</em> none</p>
<p><em>Contexts in which this component can be used:</em></p>
<ul>
<li><strong>sam-main-content</strong></li>
</ul>
<h3>SamFilterDrawerComponent</h3>
<p>The <strong>sam-filter-drawer</strong> component displays which filters have been applied with the ability to remove a single item, clear all, or save the filters. Underneath, it is an <strong>li</strong>, so each child should also be an <strong>li</strong>.</p>
<p><em>Content model:</em></p>
<ul>
<li><strong>li with sam-filter-drawer-item directive</strong></li>
</ul>
<p><em>Contexts in which this component can be used:</em></p>
<ul>
<li><strong>sam-main</strong></li>
</ul>
<h3>SamFilterDrawerItemDirective</h3>
<p>The <strong>sam-filter-drawer-item</strong> directive should be applied to <strong>li</strong> elements for each item in teh <strong>sam-filter-drawer</strong>.</p>
<p><em>Content model:</em></p>
<ul>
<li><strong>any valid HTML for li</strong></li>
</ul>
<p><em>Contexts in which this component can be used:</em></p>
<ul>
<li><strong>sam-filter-drawer</strong></li>
</ul>
<h3>SamMainContentComponent</h3>
<p>The <strong>sam-main-content</strong> component is a wrapper for the content to be displayed within the main component. It may take a <strong>sam-action-bar</strong> component, and to dispaly the content, a parent DOM node must be supplied with the <strong>samContentOutlet</strong> directive applied.</p>
<p><em>Content model:</em></p>
<ul>
<li><strong>sam-action-bar</strong></li>
<li>any HTML with <strong>samContentOutlet</strong> directive</li>
</ul>
<p><em>Contexts in which this component can be used:</em></p>
<ul>
<li><strong>sam-main</strong></li>
</ul>
<h3>SamActionBarComponent</h3>
<p>The <strong>sam-action-bar</strong> provides actions used to manipulate the main content of the page. It takes a <strong>sam-pagination-next</strong> component and additionally may take any number of additional form controls with the <strong>samPageAction</strong> directive applied.</p>
<p><em>Content model:</em></p>
<ul>
<li><strong>sam-pagination-next</strong></li>
<li>any form control with <strong>samPageAction</strong> directive</li>
</ul>
<p><em>Contexts in which this component can be used:</em></p>
<ul>
<li><strong>sam-main-content</strong></li>
</ul>
<h3>SamPaginationNext</h3>
<p>The <strong>sam-pagination-next</strong> component provides basic pagination functionality for the page.</p>
<p><em>Content model:</em> none</p>
<p><em>Contexts in which this component can be used:</em></p>
<ul>
<li><strong>sam-main-content</strong></li>
</ul>
<h3>SamPageActionDirective</h3>
<p>The <strong>samPageAction</strong> directive defines actions that can be taken to manipulate the main content of the page. This directive can be applied to any form control, and the appropriate styles and layout will be added.</p>
<p><em>Content model:</em></p>
<ul>
<li>Valid HTML for form control</li>
</ul>
<p><em>Contexts in which this component can be used:</em></p>
<ul>
<li><strong>sam-main-content</strong></li>
</ul>
<h3>SamContentOutletDirective</h3>
<p>The <strong>samContentOutlet</strong> directive may be applied to any HTML element that represents the main content of the page. This directive will appropriately style and layout the wrapper.</p>
<p><em>Content model:</em> none</p>
<p><em>Contexts in which this component can be used:</em></p>
<ul>
<li><strong>sam-page-next</strong></li>
</ul>
<h3>SamModalComponent</h3>
<p>If a modal is needed on the page, the <strong>sam-modal</strong> component should be used. However, it should not be a content child of the <strong>sam-page-next</strong> component but rather a sibling. Developers will need to configure that for themselves.</p>
<p><em>Content model:</em></p>
<ul>
<li>See <strong>sam-modal</strong></li>
</ul>
<p><em>Contexts in which this component can be used:</em></p>
<ul>
<li>Any Angular application</li>
</ul>
