<h1>List/Result Page</h1>
<p>The List/Result page is used when the user is presented with a listing of related items (such as a Workspace Tier 2 page) or search results.</p>
<ul>
<li>The page has controls enabling the user to filter or sort items on the list</li>
<li>The page has no need to be broken into distinct sections due to its length or complexity</li>
</ul>

<h2>List Page Elements</h2>
<h3>Functional 'Add' button (optional)</h3>
<p>Depending on role access and domain functionality, a functional button to add a new item to the list may be available.</p>
<h3>Filter (optional)</h3>
<p>Filter controls are organized on the left-hand side based on priority or organized based on workflow (such as for Wage Determination).</p>
<h3>Section Search Bar</h3>
<p>In the upper right is a Search field, which returns results specific to the current site section.</p>
<h3>Display count</h3>
<p>The display count provides a count of displayed results and total results.</p>
<h3>Sort control</h3>
<p>The sort control enables the user to adjust the order in which list items appear.</p>
<h3>Pagination</h3>
<p>Pagination indicates which page of results a user is on and enables the user to traverse to additional pages.</p>
<h2>List Item Elements</h2>
<p>List Items can display a variety of data depending on business needs. Following are the most common data elements that allow the user to easily identify and act on a list item.</p>
<ul>
<li>Item Name</li>
<li>Item Description</li>
<li>Status</li>
<li>Action Buttons (e.g., Edit, Delete, Archive)</li>
<li>Dates (e.g., Published, Due, Modified)</li>
</ul>
