Workspace Pages

  1. WorkspaceLanding Page

    A workspace landing page provides an entry point for multiple independent categories of information. It is customized to reflect the type of work done by an individual. For example, the illustration to the left might be for someone create, edits, or oversees work with pears, bananas, monkeys, apples, and oranges. There is a workspace element for each type of information (pears, bananas, monkeys, etc) that is the start place for working with that information. It is important to help someone see quickly which categories or records require their attention or action.

  2. Assistance ListingsWorkspace Page

    A workspace page provides an entry point for managing a single category of information. It may be customized to reflect a person’s role in managing that information (contributor, author, reviewer, etc). A workspace page typically lists the records a person manages, provides searching, sorting, and filtering of those records, infographics of key summary metrics, and status information. It is important to help someone quickly see which records require their attention or action.

Data Entry Pages

  1. Role RequestSingle Page Editor

    A single page editor has a small number of fields, often completed and submitted in a single session.

  2. Assistance ListingMultiple Page Editor

    Data entry is split across multiple pages where there are a large number of fields to be completed. We can expect that users may fill out some multi-page forms in one session and other may be completed across multiple sessions.

  3. Assistance ListingReview Page

    A review page may be provided during data entry to accompany a multi-page form, giving people a view of all fields on a single page. The fields will be read only and displayed concisely - not using the real estate that an editor might require. A review page typically has fields with restricted access which are visible to people who edit and perform reviews. If all fields are public and there is a “Published View,” a review page may be redundant.

  1. Assistance ListingVersion Page

    A version page is a review page that shows people what has changed since a previous version of the record was published.
